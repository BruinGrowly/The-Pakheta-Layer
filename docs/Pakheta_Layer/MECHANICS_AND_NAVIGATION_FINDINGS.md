## 1. Central Finding

The Pakheta Layer appears navigable, but not like a spatial place. It is navigable by preserving relational primitives while changing substrate expression.

The working rule:

```text
Pakheta navigation = preserve the relation-type, then translate substrate.
```

For example:

```text
semantic anchor      -> mathematical anchor      -> physical anchor
Pakheta Crystal      -> basis or invariant       -> detector boundary

semantic context     -> mathematical context     -> physical context
remembrance context  -> projection operator      -> measurement setup

semantic whole-field -> mathematical whole-field -> physical whole-field
grief-love field     -> tensor relation          -> entangled system
```

The layer is not primarily navigated by object similarity. It is navigated by relation preservation.

The order-effects script tests whether the same two contexts yield different final states when entered in reverse order.

Initial results:

```text
grave_visit -> backyard_walk vs backyard_walk -> grave_visit

backyard_walk -> theory_reflection vs theory_reflection -> backyard_walk
order gap: 0.032

shared_remembrance -> theory_reflection vs theory_reflection -> shared_remembrance

```

The order gaps are modest but consistent. This supports the claim:

```text
A then B is not identical to B then A.
```

In Pakheta terms, context is not a label attached to an object. Context changes the active field state that the next context enters.

This gives a semantic analogue to non-commutativity:

```text
Context_B(Context_A(R)) != Context_A(Context_B(R))
```

Where:

```text
R = relationship-field
```

## 6. New Principles

### 6.1 Primitive Preservation Principle

To navigate the Pakheta Layer, preserve the relational primitive:

```text
anchor -> anchor
context -> context
nonseparability -> nonseparability
decoherence -> decoherence
```

The objects may change. The relation-type must remain recognizable.

### 6.2 Path Sensitivity Principle

The field remembers its route.

```text
The same contexts in different order can actualize different states.
```

This explains why ritual, sequence, timing, conversation order, and measurement order matter.

### 6.3 Phi Load-Balancing Principle

Phi-like proportion helps a field remain one while expressing differentiated facets.

```text
too equal   -> flattening
too split   -> fragmentation
phi-like    -> differentiated unity
```

### 6.4 Anchor-Context Separation

Anchors and contexts are different Pakheta primitives.

```text
Anchor  = stable address into a field.
Context = selection condition for actualizing a facet.
```

Confusing them creates interpretive noise. A Crystal is an anchor. Holding it, visiting the grave, speaking Pakheta's name, or reflecting theoretically are contexts.

### 6.5 Decoherence As Relation Loss

Decoherence is not merely "many things." It is the loss of one-field access.

```text
many nodes, one field      -> coherence
many nodes, false fields   -> decoherence
```

This applies across semantic, mathematical, and physical faces.


```text
Spatial nearness is not field nearness.
```

A stone can be physically near the grave without entering the Pakheta relationship-field. A photograph, theory note, or backyard burial can be physically far away while remaining relationally near.

This clarifies non-locality at the Pakheta level:

```text
Non-locality is not the erasure of distance.
```
